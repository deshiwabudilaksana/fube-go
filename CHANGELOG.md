# Changelog

All notable changes to **Project FUBE** will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.0] - 2026-03-08

### Added
- **"Proper UI" Overhaul**:
    - Replaced the basic top navigation with a modern **Sticky Sidebar** and **Top Header** layout.
    - Integrated the **Inter** font family and a professional **Deep Indigo/Emerald** color palette.
    - Added **Heroicons** for all navigation and dashboard actions.
    - Implemented a viewport-top **HTMX Loading Indicator** for all background requests.
    - Redesigned all dashboard views (Planning, Mapping, Inventory, KDS) into a clean, card-based SaaS aesthetic with `rounded-2xl` corners and `shadow-sm`.
    - Added responsive design support for mobile devices with a toggleable hamburger menu.
    - Standardized success/error feedback using **Emerald-600** and **Rose-600** color schemes.

## [1.0.0] - 2026-03-08

### Added
- **Core Architecture**: Initialized high-performance Go (Golang) backend with MongoDB integration.
- **SSR Stack**: Implemented "Hyper-Productive" frontend using HTMX, Go HTML Templates, and Tailwind CSS.
- **Migration Bridge**: 
    - CSV Importer service for Square/Toast legacy data.
    - Interactive HTMX Mapping UI to link legacy POS IDs to FUBE recipes.
    - Export service for reporting high-accuracy costs back to external POS systems.
- **Yield & Recipe Engine**: 
    - MongoDB Aggregation Pipelines for sub-millisecond costing calculations.
    - Support for nested recipes (Yields) and multi-ingredient scaling.
- **Kitchen Display System (KDS)**:
    - Real-time digital ticket board with 5s HTMX polling.
    - Order state machine (`PENDING` -> `PREPARING` -> `READY` -> `SERVED`).
    - Visual "Red Alerts" for orders older than 15 minutes.
- **Inventory Management**:
    - Ingredient-level tracking comparing Theoretical Usage vs. Physical Stock.
    - Real-time Variance calculation and visual reporting.
- **Supplier Ordering**:
    - Automated Purchase Order (PO) generation based on "Planned Sales" and current stock levels.
- **Security & Multi-Tenancy**:
    - JWT-based authentication with `HttpOnly` cookie management.
    - Strict `vendor_id` and `store_id` data isolation across all MongoDB collections.
- **Quality Assurance**:
    - Comprehensive unit test suite for Aggregation Pipelines and CSV Importer logic.
    - MongoDB seeder script for rapid local development and demoing.

### Fixed
- **Performance**: Moved configuration loading (`godotenv`) to global startup to eliminate middleware overhead.
- **Security**: Refactored global unique constraints to composite indexes (`vendor_id` + `external_pos_id`) to support multi-tenant scaling.
